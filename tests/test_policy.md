# テスト方針

## 1. 目的・対象範囲・テストレベル

### 目的
本プロジェクトのコードの品質を担保し、デプロイや機能追加に伴うデグレード(先祖返り)を防止することを目的とします。

### 対象範囲
- `src/streamlit_cloudflare_d1/model.py` (データモデル定義)
- `src/streamlit_cloudflare_d1/main.py` (アプリケーションのロジック部分)

### テストレベル
- **単体テスト (Unit Test)**: 主に SQLAlchemy モデルの CRUD 操作の正当性を検証します。

## 2. ローカルおよびCIでの実行手順

### ローカルでの実行手順
テスト実行前に、ローカル環境でモック用D1サーバー（ポート `8787`）があらかじめ起動している必要があります。
起動した状態で、仮想環境上で以下のコマンドを実行してテストを実行します。
```bash
uv run pytest
```

### CI (継続的インテグレーション) での実行手順
- 現時点ではCIは未定です。導入する際は、GitHub Actions 等のCI環境で `uv run pytest` を稼働させる設定を行います。

## 3. カバレッジ測定方法と使用ツール

### 使用ツール
- `pytest-cov` (カバレッジ測定用プラグイン)

### 測定手順
ローカルで以下のコマンドを実行し、カバレッジおよび未カバーの行を確認します。
```bash
uv run pytest --cov=src --cov-report=term-missing
```

### 目標値
- 行カバレッジ(line coverage)で **80%以上** を目標とします。

## 4. カバレッジが80%未満だった場合の処置手順

1. カバレッジレポート (`--cov-report=term-missing` の出力結果) を確認し、未カバーのファイルと行範囲を特定します。
2. 優先度が高いと判断される未カバーの関数や条件分岐 (特にビジネスロジックを含む箇所) に対して、テストケースを追加します。
3. テスト追加後、再度カバレッジを測定して80%以上に到達したことを確認します。
4. どうしても80%以上に達しない場合は、未カバー箇所の理由 (外部サービス依存等でテストが困難な理由) をドキュメント化し、ユーザーに提案・相談します。
